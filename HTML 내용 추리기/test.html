Git이란 ?

○ 2005년 리누스 노발즈가 만든 분산형 버전 관리 시스템(DVCS)

  • 여러 개발 PC와 저장소에 분산해서 저장함

  • 컴퓨터에 있는 대부분의 파일을 Git으로 관리하는 것이 가능함


○ 특징?

  • 코드를 저장하거나 저장 지점으로 되돌아가게 함
  • 가장 많이 사용, 무료 및 오픈 소스
  • 빠르고 오프라인 작업 가능 및 실수 고치기 쉬움
  • 효율적 협업 가능 및 개인의 포트폴리오 관리 가능

○ Git 설치

  # 터미널에서 명령을 입력한다.

  • ~$ sudo apt update
  • ~$ sudo apt install git
  • ~$ git

○ 로컬 저장소 생성

  # "Projects/사용자 이름/git_test" 이름으로 생성

  • ~$ mkdir -p ~/Projects/erica/git_test
  • ~$ cd ~/Projects/erica/git_test
  • ~$ git init (로컬 저장소 초기화 + 마스터 브랜치 생성)

  Tip) ~$ rm -rf .git (Git 로컬 저장소 삭제)


○ 정보 등록

  # 특정 사용자 : --global

  # 시스템 전체 : --system

  # 특정 저장소 : --local

  # 사용자의 이름과 이메일 등록

  • ~$ git config --global user.email 사용자 이메일 주소
  • ~$ git config --global user.name 사용자 이름

  Tip) ~$ git config --list (전체 설정 확인)


○ 첫 번째 커밋하기

  # 스테이징 (Staging)

  • git add 명령 사용
  • 커밋할 파일 선택해서 모아두는 장소 (Stage Area)

  # 커밋 (Commit)

  • "소프트웨어 버전" or "버전 만들기"
  • git commit 명령 사용
  • 현재 커밋을 가리키는 HEAD 생성
  • 스테이지 영역의 파일을 로컬 저장소(.git)으로 전달

  # Working directory에 파일 생성

  • ~$ echo "hello world" > a.txt
  • ~$ echo "hello world" > b.txt
  • ~$ echo "hello world" > c.txt
  • ~$ git status # 현재 파일 상태 확인

  # 커밋을 위한 준비(Staging)

  • ~$ git add a.txt
  • ~$ git status
  • ~$ git add *
  • ~$ git status

  # 파일 수정

  • ~$ echo 'file modified' >> a.txt
  • ~$ git status

  # Working directory의 변경사항 되돌리고 파일 커밋

  • ~$ git checkout a.txt
  • ~$ git commit -m "first commit"

  # 변경 이력 확인 - git log

  • ~$ git log

○ 두 번째 커밋하기

  # Working directory의 파일 수정과 스테이징

  • ~$ echo 'file modify' >> a.txt
  • ~$ git add a.txt
  • ~$ git status

  # 두 번째 커밋 실행

  • ~$ git commit -m "second commit"

  # 커밋 이력 확인

  • ~$ git log

  # 해시 값 사용하여 다른 커밋으로 이동

  • ~$ git checkout 6문자 (first commit의 앞 6문자)

  # 변경사항 확인하고 원래대로 되돌리기

  • ~$ git log
  • ~$ git switch -

  # 커밋 이력을 통해 원래대로 되돌아감을 확인

  • ~$ git log

○ 원격 저장소에 커밋 올리기

  # 로컬 저장소에서 만들었던 커밋들을 원격 저장소로 보내는 작업

  • ~$ git remote add origin https://github.com/계정 ID/원격저장소 이름.git

  # push 명령으로 원격 저장소에 업로드

    최근에는 암호 방식을 지원하지 않아, 토큰을 입력해야한다.
  • ~$ git push origin -u master

○ 원격 저장소 내려받기

  # 클론(Clone)

    이전 버전과 원격 저장소의 주소 등이 내 컴퓨터에 저장

    내 컴퓨터에 내려받을 폴더 생성 : [git_test_clone]

  • ~$ mkdir ~/Projects/erica/git_test_clone
  • ~$ cd ~/Projects/erica/git_test_clone

  # Github 원격 저장소에서 [Code/Clone] 항목에서 URL 주소 복사

    "git clone" 명령 뒤에 복사한 주소를 붙여 넣는다.
  • ~$ git clone https://github.com/이름/저장소.git
  • ~$ ls -al # git clone 폴더에 로컬 저장소 폴더가 보이면 성공

○ 내려받기한 저장소에 커밋하기

  # 새 파일 작성

    README.md 파일 작성
  • ~$ vi README.md # vim 에디터 사용

  # 세 개의 명령으로 새로운 파일 푸시

  • ~$ git add README.md
  • ~$ git commit -m "Contributors List"
  • ~$ git push origin master
    • 이후 페이지 새로 고침하여 푸시한 커밋 확인

○ 로컬 저장소 갱신하기

  # [git_proj] 저장소 갱신

  • ~$ cd ~/git_test
  • ~$ git pull origin master
    • ls 명령으로 변경 사항 확인
